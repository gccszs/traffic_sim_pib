# 修复地图路径问题

## 问题
数据库中存储的是 Python 服务端的绝对路径，导致分布式部署和路径变化时失效。

## 修复方案

### 1. 修改 Python gRPC 服务端（`map_convert_services/grpc_server.py`）
- 修改 `_convert_file` 方法，返回相对路径而非绝对路径
- 返回格式：`{userId}/{xmlFileName}`

### 2. 修改 Python 仿真服务（`map_convert_services/simulation_service.py`）
- 修改 `CreateSimeng` 方法，正确处理传入的路径
- 支持相对路径和文件名两种格式

### 3. 验证 Java 端代码
- 确认 Java 端正确保存和传递相对路径
- 无需修改现有代码逻辑

## 预期效果
- 路径与部署位置解耦
- 支持分布式部署
- 提高系统可移植性
